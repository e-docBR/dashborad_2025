# Contribuindo para o ColaboraEdu

Obrigado pelo interesse em contribuir para o ColaboraEdu! Este documento fornece diretrizes para ajudá-lo a começar.

## Código de Conduta

Por favor, mantenha um ambiente respeitoso e colaborativo. Seja gentil e inclusivo com todos os participantes do projeto.

## Como Contribuir

### Reportando Bugs
Se você encontrar um bug, por favor abra uma Issue descrevendo:
1.  Passos para reproduzir o problema.
2.  Comportamento esperado vs. comportamento real.
3.  Ambiente (Navegador, SO, etc.).

### Sugerindo Novas Funcionalidades
Abra uma Issue com a tag `enhancement` descrevendo sua ideia. Explique por que ela seria útil e como poderia funcionar.

### Desenvolvimento

1.  **Fork e Clone**: Faça um fork do repositório e clone para sua máquina local.
2.  **Branch**: Crie uma branch descritiva para sua tarefa: `git checkout -b feature/nome-da-feature` ou `fix/nome-do-bug`.
3.  **Padrões de Código**:
    *   Utilizamos TypeScript. Por favor, mantenha a tipagem forte.
    *   Siga as convenções de estilo do projeto (Prettier/ESLint já configurados).
    *   Componentes de UI devem seguir o padrão do shadcn/ui.
4.  **Commits**: Escreva mensagens de commit claras e objetivas.
5.  **Pull Request**: Envie suas alterações via Pull Request, referenciando a Issue relacionada (se houver).

## Estrutura de Pastas Importante

*   `src/app`: Rotas da aplicação.
*   `src/components/ui`: Componentes de UI genéricos (botões, cards, etc.).
*   `src/lib/ai`: Lógica relacionada aos serviços de Inteligência Artificial.

Obrigado pela sua ajuda!
